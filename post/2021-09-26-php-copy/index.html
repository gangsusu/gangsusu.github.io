php中的深拷贝和浅拷贝

2021年9月26日
| php

深拷贝和浅拷贝的通俗理解

  1. 深拷贝：在赋值的时候，完全复制，是完全不同的两个值，对其中一个修改，不会影响到另一个。
  2. 浅拷贝：在赋值的时候，是引用赋值，对应的是同一个值，对其中一个修改，另一个的值也会随之更改。

在php中，常用的数据数据类型如int和string，是深拷贝，直接复制了一个新的出来，两个值之间互不干涉，这个比较简单，不做赘述。我们来看一下，稍微有点不同的对象类型。

代码验证

  1. 首先，我们来看一看最简单的，只包含一个普通属性的情况：
class TestOne
{
    public $data = 11;
}

$a = new TestOne();
$b = $a;
$a->data = 22;
var_dump($a->data);   // 22
var_dump($b->data);   // 22

我们可以看到其中一个值的改变，另一个值也随之改变了，说明了不同于php常见数据类型的深拷贝，这里发生的浅拷贝。

  1. __clone可以复制，我们看看上面的情况用__clone会如何
class TestTwo
{
    public $data = 11;
}

$c = new TestTwo();
$d = clone $c;
$c->data = 22;
var_dump($c->data);   // 22
var_dump($d->data);   // 11

我们可以看到，一个值得改变没有影响到另一个，发生的是深拷贝。

  1. 我们看一看更复杂的情况，加上对象属性会怎么样
class Test
{
    public $a = 1;
}

class TestThree
{
    public $b = 1;
    public $obj;

    public function __construct()
    {
        $this->obj = new Test();
    }
}

$e = new TestThree();
$f = $e;
$e->b = 2;
$e->obj->a = 3;
var_dump($e->b); //2
var_dump($e->obj->a); //3
var_dump($f->b); //2
var_dump($f->obj->a); //3

我们可以看到，普通属性和对象属性操作的值都该改变了，说明都发生了浅拷贝，这和TestOne中是一样的。

  1. 我们用__clone来试一下
class TestFour
{
    public $b = 1;
    public $obj;

    public function __construct()
    {
        $this->obj = new Test();
    }
}

$g = new TestFour();
$h = clone $g;
$g->b = 2;
$g->obj->a = 3;
var_dump($g->b); //2
var_dump($g->obj->a); //3
var_dump($h->b); //1
var_dump($h->obj->a); //3

发生了一些不同，普通属性发生的是深拷贝，但是对象属性操纵的值发生的还是浅拷贝。

  1. 那我们也想让它深拷贝的彻底一点，怎么做呢？
class TestFive
{
    public $b = 1;
    public $obj;

    public function __construct()
    {
        $this->obj = new Test();
    }

    public function __clone()
    {
        $this->obj = clone $this->obj;
    }
}

$i = new TestFive();
$j = clone $i;
$j->b = 2;
$j->obj->a = 3;
var_dump($j->b); //2
var_dump($j->obj->a); //3
var_dump($i->b); //1
var_dump($i->obj->a); //1

我们重写一下__clone方法，当clone的时候，就完成了对象属性的拷贝。

  1. 上面的操作有点麻烦，其实还有其他的方法。
class TestSix
{
    public $b = 1;
    public $obj;

    public function __construct()
    {
        $this->obj = new Test();
    }
}

$m = new TestSix();
$n = unserialize(serialize($m));
$m->b = 2;
$m->obj->a = 3;
var_dump($m->b); //2
var_dump($m->obj->a); //3
var_dump($n->b); //1
var_dump($n->obj->a); //1

我们做一次对象的序列化和反序列化，也可以做到深拷贝。

总结

php中普通的数据类型，赋值发生的是深拷贝，对象发生的是浅拷贝。想要对象的深拷贝，就可以按照上面的方法来进行深拷贝。go语言中不同的数据类型，在赋值的时候，根据数据类型的不同，是发生深拷贝还是浅拷贝，也不尽相同，下一篇我们一起来看一下go语言中的深拷贝和浅拷贝。
