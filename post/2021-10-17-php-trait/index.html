trait的使用

2021年10月17日
| php

简单介绍

trait是php5.4之后新增的一个功能，可以将某些需要反复使用的代码封装到trait中，以达到代码复用的效果。 它的本质不是类，所以不能被实例化，可以将它理解为一个代码片段，可以引入来进行方法的复用。

简单使用

trait t1
{
    public $name = 'tom';

    public function printName()
    {
        echo 'my name is ' . $this->name . PHP_EOL;
    }
}

class c1
{
    use t1;
}

$obj = new c1();
$obj->printName(); // my name is tom

这就是最简单的用法了，定义好一个trait，在需要用的类里使用use来引入，就可以复用trait里的方法了。

use多个trait

trait可以解决php单继承的问题，可以use多个trait

trait hello
{
    public function sayHello()
    {
        return 'hello';
    }
}

trait world
{
    public function sayWorld()
    {
        return 'world';
    }
}

class helloWorld
{
    use hello, world;
}

$obj1 = new helloWorld();
var_dump($obj1->sayHello() . ' ' . $obj1->sayWorld()); //hello world

优先级

如果在类中有继承，也有trait的使用，当继承类和被继承类以及trait中有重名的方法时，它的优先级是：本类的优先级>trait的优先级>基类的优先级。 可以自行进行测试。

引入多个trait之中重名方法的处理

当引入的多个trait中有重名的方法时候，就会引起冲突，可以使用insteadof来定义优先级。

trait t1
{
    public function eat()
    {
        echo 't1 eat' . PHP_EOL;
    }

    public function run()
    {
        echo 't1 run' . PHP_EOL;
    }
}

trait t2
{
    public function eat()
    {
        echo 't2 eat' . PHP_EOL;
    }

    public function run()
    {
        echo 't2 run' . PHP_EOL;
    }
}


class cat
{
    use t1, t2 {
        t1::eat insteadof t2;
        t1::run insteadof t2;
    }
}

$catObj = new cat();
$catObj->eat(); //t1 eat
$catObj->run(); //t1 run

trait的嵌套

trait hello
{
    public function sayHello()
    {
        return 'hello';
    }
}

trait world
{
    public function sayWorld()
    {
        return 'world';
    }
}

trait hello_world{
    use hello,world;
}

class helloWorld
{
    use hello_world;
}

$obj1 = new helloWorld();
var_dump($obj1->sayHello() . ' ' . $obj1->sayWorld()); //hello world

更改方法的访问控制

我们可以在use某个trait的时候，更改方法的访问属性。

trait hello
{
    public function sayHello()
    {
        return 'hello';
    }
}

trait world
{
    public function sayWorld()
    {
        return 'world';
    }
}

trait hello_world{
    use hello,world{
        sayWorld as private;
    }
}

class helloWorld
{
    use hello_world;
}

$obj1 = new helloWorld();
var_dump($obj1->sayHello() . ' ' . $obj1->sayWorld());

因为我们修改了sayWorld方法为private属性，将无法调用该方法，会抛出一个错误。

使用场景

我个人在开发的时候，会将一些全局的轻便的一些代码段和功能作为一个trait来使用，因为它可能不属于任何你的抽象类，定义在任何被继承的抽象类里都不合适。在laravel框架中，也使用了很多trait,如：

<?php

namespace App\Http\Controllers;

use Illuminate\Foundation\Bus\DispatchesJobs;
use Illuminate\Routing\Controller as BaseController;
use Illuminate\Foundation\Validation\ValidatesRequests;
use Illuminate\Foundation\Auth\Access\AuthorizesRequests;

class Controller extends BaseController
{
    use AuthorizesRequests, DispatchesJobs, ValidatesRequests;
}

在框架的这个基础控制器中，使用了认证，job分发，验证的trait。
